# Security Policy

Thank you for helping to keep BadSMTP secure.

If you believe you have found a security vulnerability in BadSMTP, please report it to us as follows.

Contact
-------
Send an email to [badsmtp@badsmtp.com](mailto:badsmtp@badsmtp.com).

Please do not open public issues with vulnerability details â€” use the contact address so we can coordinate disclosure.

What to include
---------------
To help us triage and reproduce the issue quickly, please include as much of the following information as you can:

- A short, clear summary of the vulnerability and its impact
- Product version(s) affected and how you built/obtained the binary
- Steps to reproduce the issue (a minimal PoC is extremely helpful)
- Any relevant logs, configuration files, or network captures (redact any sensitive data)
- Your contact information so we can follow up with questions

Encryption
----------
If you prefer to send sensitive information encrypted, indicate this in your initial email and we will provide a public key or alternate secure channel.

What we will not do
-------------------
- We will not publish exploit code or specific PoCs without first coordinating with affected parties.
- We will not pursue legal action against security researchers who follow this policy and act in good faith.

Credit
------
We welcome credit for responsible disclosure. If you want acknowledgement in release notes or project documentation, say so in your report. If you prefer to remain anonymous, let us know and we will respect your choice.

Thank you for helping make BadSMTP safer.
